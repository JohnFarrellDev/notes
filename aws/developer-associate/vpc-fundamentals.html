VPC and Subnets

VPC is a virtual private cloud, this is a private network within AWS that allows deployment of resources within the VPC. VPCs are regional based, different regions have separate VPCs

Subnets allow you to partition your network within your VPC, subnets are an availability zone resource.

A subnet can be public so can access the WWW and be accessed by the WWW.

A subnet can also be private so the subnet is not internet accessible.

To define access to the internet and between subnets we use Route Tables.

Internet Gateways and NAT Gateways

Our internet gateway allows our subnets to connect to the internet, the public subnet has a connection to the IGW and the IGW can connect to the internet.

If we want our private subnets to access the internet we can use NAT gateways which are managed by AWS. This allow instances in the private subnet to access the internet while remaining private.

The NAT gateway is deployed within our public subnet, the NAT gateway has a route to the IGW. The private subnet then connects to the NAT gateway in the public subnet.

Network ACL and Security Groups

Network ACL (access control list) is a firewall which controls traffic to and from the subnet.

The NACL can have ALLOW and DENY rules that are attached at the subnet level, rules only include IP addresses.

We also have security groups that can control traffic to an ENI (elastic network interface) which in turn controls traffic to our EC2 instances. The security group can only have ALLOW rules and reference IP addresses or other security groups.

VPC Flow Logs

Captures all information about IP traffic going into your interfaces.

  • VPC flow Logs
  • Subnet flow logs
  • Elastic network interface flow logs

Helps monitory and troubleshoot connectivity issues.

Captures network information from AWS managed interfaces too (ELB, ElastiCache, RDS, etc...)

VPC Flow Logs data can be sent to S3, CloudWatch, Kinesis Data Firehose.

VPC Peering

Connect two VPCs in different cloud environments or different regions using AWS' network. This allows them to communicate and behave as if they are in the same network.

The IP ranges for each VPC CIDR cannot overlap!

VPC peering connection is not transitive.

VPC Endpoints

VPC endpoints allow us to connect to AWS services using a private network instead of the public web. This gives ue enhanced security and lower latency to access AWS services.

Sit to Site VPN

Allows us to connect on premise VPN to AWS.

Data is automatically encrypted over the public internet.

Direct Connect (DX)

Established a physical connection between on-premises and AWS.

The connection is private, secure and fast but takes at least a month to establish.
