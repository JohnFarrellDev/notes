Getting Started With AWS

AWS launched when Amazon realized they had all this technical skill and extra computing power and they could offer this as a service to other companies. Over the years this has continuously expanded into the huge offerings now supported by AWS and their services being utilized by almost every large company.

In 2019 AWS had $35 billion in revenue and accounted for 47% of the cloud market. There are over 1,000,000 active users on AWS.

AWS enables people to build sophisticated scalable applications. The uses apply to a diverse set of industries such as enterprise IT, data storage and back up, big data analytics, website hosting, mobile apps and so on.

AWS Global Infrastructure

AWS is a global service with servers hosted all around the world, this global service is made of of Regions, Availability Zones, Data Centers, Edge Locations and Points of Presence.

Global Infrastructure Map

We can see that in a continent such as America there are multiple regions such as us-east-1, us-east-2, us-east-3. Within each region there are multiple Availability Zones too.

Regions and Availability Zones

Most services are region scoped (excluding IAM, Route 53, CloudFront, WAF).

Choosing which region to launch an application can depend on multiple factors:

  • Compliance with data governance laws
  • Proximity to customers to reduce
  • latency
  • Now all regions contain every service
  • Pricing can vary between regions

Regions consist of AZs. Most regions have 3 AZs, the lowest is 2 and the max is 6. Availability zones are a cluster of logical data centres.

Each availability zone has independent power, cooling and physical security and is connected via redundant, ultra-low latency networks. The AZs are isolated so one AZ going down should not impact the other AZs. By deploying to multiple AZs services will have greater fault tolerance. AZs belonging in the same region are interconnected with high-bandwidth, low-latency networking. Traffic between AZs is encrypted. The network performance is sufficient to accomplish synchronous replication between AZs. Within the AZ there can be multiple data centres each with their own redundant power, networking and connectivity.

Edge Locations - AWS Points of Presence

AWS currently have over 400 points of presence, 400+ edge locations + 10 regional caches in over 90 cities over 40+ countries.
